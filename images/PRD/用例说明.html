用例说明

用例名称

查看卡号密码

用例说明

电子卡订单用户购买成功后查看订单中的卡号密码以及面额信息

用例作用范围

饭卡线上电子卡订单

优先级

must to have

主要用户

购买电子饭卡的用户

涉众

饭卡售卖系统

前置条件

电子卡订单已经成交

触发条件

用户主动进入订单列表或者订单详情页查看卡号密码

主要流程

1、用户进入饭卡订单列表页或者订单详情页，选择查看卡信息

2、检查验证状态是否通过，若已通过，则直接进入查看卡信息页面，若未通过，则检查当日查看该笔订单卡号密码验证短信发送是否超过十条（该条数与快速领用共用），若未超过，则发送验证短信到订单中填写的手机号

3、用户接受短信并填写验证码

4、校验验证码的正确性，若正确，则进入查看卡信息页面

分支流程

2A1、若当日查看该笔订单卡号密码的验证短信发送次数已经超过十条，则提示用户：当日短信发送过于频繁，请明天重试

3A1、若用户验证码错误，统一验证码错误前四次，可以重新尝试输入，若同一验证码第五次错误，该条验证码失效，并发送一条新的短信验证码（如果该条短信为当日第十一条短信，也无法发送，提示内容同上）

异常流程

无

后置条件

查看卡号密码后，30分钟内可以通过连接进入快速领用，并跳过验证流程

业务规则

验证码通过验证后，有效时长为30分钟，超时之后验证状态失效，需要重新验证，并且验证状态与快速领用共用

约束

1、验证短信每日最多发送十条的限制仅针对同一笔订单，多笔订单可以超过十条短信

2、每条短信验证码有效时长为30分钟，从发送后开始计算，超时后验证码失效

备注

查看卡号密码的验证状态可以和快速领用共用，但不能和确认收货的验证状态共用

用例名称

制卡（常规库存）

用例说明

用于系统生成饭卡卡号、密码并将实体卡制成实体卡流程

用例作用范围

饭卡电子卡、实体卡生成

优先级

must to have

主要用户

制卡厂

涉众

制卡系统、库存系统

前置条件

无

触发条件

库存中已生成的卡号密码少于等于1W张

主要流程

1、库存中已经生成的卡号密码且还没卖出的饭卡数量少于等于1W张时，一次性生成10W张饭卡的卡号密码（密码需要加密保存）

2A、若库存为电子卡，则将其直接标记为可售卖饭卡

分支流程

2B1、若该库存为实体卡，卡号密码生成完毕后，系统自动将卡号密码加密，并发送至制卡厂

2B2、制卡厂完成制卡后，将制卡完成信息反馈给饿了么

2B3、将本批次实体卡标记为可售卖饭卡

异常流程

1、生成卡号券密异常时，发送报警邮件至yinghao@ele.me和renmin.chen@ele.me

2、实体卡将卡号密码发送给制卡厂N天后若没有反馈，需要发送报警邮件至yinghao@ele.me和renmin.chen@ele.me，后续人工处理（N的值视谈判结果而定）

后置条件

开始售卖饭卡

业务规则

无

约束

无

备注

无

用例名称

购买饭卡

用例说明

用户外网购买饿了么饭卡

用例作用范围

用户在手机APP或者H5页面购买饿了么饭卡

优先级

must to have

主要用户

饿了么用户

涉众

售卖系统、饿了么用户

前置条件

可售卖库存足够

触发条件

进入饭卡购买首页

主要流程

1、进入饭卡售卖首页

2、选择产品面额和张数

3、判断用户是否登录，若已登录，直接进入订单信息填写页

4、填写订单信息，根据不同的订单情况变化订单总额

5、创建订单（订单号生成规则见交互稿中说明），订单状态为：支付中

6、支付订单饭卡

7、支付完成后提示订单提交完成

分支流程

2A1、若用户未登录，则需要跳转至登录/注册页面

3A1、用户可以选择是否需要发票、填写配送地址和联系人信息

3A2、若用户购买订单总额大于等于1000元（不含配送费），则需要填写联系人信息，并将该信息用作购买人和持有人信息

异常流程

1、订单在收到支付平台确认的“支付成功”信息前，订单状态统一为：支付中

2、订单创建失败时需要重试，重试三次失败需要提示用户“网络异常，稍后重试”

后置条件

1、订单支付成功后，若为电子卡，发送短信告知用户前往订单中心查询卡号密码；若为实体卡，则订单状态改为“处理中”

2、若用户需要发票，则将发票信息传给财务部，由财务部开具相应发票

业务规则

1、订单总额大于等于￥1000（不含配送费）需要填写订单购买人信息

2、订单总额大于等于￥XXXXXX，免收配送费

约束

未登录用户无法购买饭卡

备注

无

用例名称

确认收货

用例说明

实体卡用户订单成交后确认收货才能激活饭卡进行领用

用例作用范围

实体卡订单，饭卡配送后

优先级

must to have

主要用户

购买饭卡的客户

涉众

饭卡购买客户、短信通道、售卖系统

前置条件

用户饭卡订单状态为“配送中”

触发条件

后台运营人员开始配送并将配送编号填写至后台模块中

主要流程

1、用户进入饭卡订单列表页或订单详情页，选择确认收货

2、检查验证状态是否通过，若已通过，则直接进入查看卡信息页面，若未通过，则检查当日确认收货该笔订单验证短信发送是否超过十条，若未超过，则发送验证短信到订单中填写的手机号

3、用户接受短信并填写验证码

4、校验验证码的正确性，若正确，则进入确认页面

5、用户确认订单中的饭卡已经收到

分支流程

2A1、若当日确认收货的验证短信发送次数已经超过十条，则提示用户：当日短信发送过于频繁，请明天重试

4A1、若用户验证码错误，统一验证码错误前四次，可以重新尝试输入，若同一验证码第五次错误，该条验证码失效，并发送一条新的短信验证码（如果该条短信为当日第十一条短信，也无法发送，提示内容同上）

异常流程

无

后置条件

确认收货后，激活该订单下所有饭卡，用户可以进行领用

业务规则

验证码通过验证后，有效时长为30分钟，超时之后验证状态失效，需要重新验证

约束

无

备注

确认收货的验证状态不可跟查看卡号密码以及快速领用的验证状态共用

3、用户输入卡号密码进行领用（一次领用一张）

4、校验用户领用功能是否被锁定，若没被锁定，则继续校验卡号密码的正确性，若领用功能已被锁定，提示用户，卡号密码输入错误次数过多，半小时后重试

5、校验卡号密码是否正确，若正确，将该饭卡关联到该账号下，并提示用户领用成功

分支流程

2A1、若用户未登录，则跳转至登录/注册页面

2A2、登录完毕后直接进入领用页面

5A1、用户输入的卡号和密码校验不正确时，查询输入错误次数是否小于等于5次，若是，则提示用户输入错误并可以重新输入，若连续输错次数大于5次，则锁定领用功能30分钟

异常流程

无

后置条件

领用成功后需要在收支明细中增加记录，交易类型为：用户领用，时间为：领用时间（精确到“天”），金额为：领用金额

业务规则

无

约束

无

备注

无

1、用户主动进入订单列表或者订单详情页快速领用

主要流程

1、用户进入饭卡订单列表页或者订单详情页，选择【快速领用】；或者在用户查看卡号密码后选择继续【快速领用】

2、检查验证状态是否通过，若已通过，则直接进入快速领用页面，若未通过，则检查当日领用该笔订单中饭卡的验证短信发送是否超过十条（该条数与查看卡号密码共用），若未超过，则发送验证短信到订单中填写的手机号

4、校验验证码的正确性，若正确，则进入查看卡信息页面

5、用户选择希望领用的饭卡点击【领用】按钮进行领用，或者点击【全部领用】按钮领用该笔订单下所有未领用的饭卡至该账户

分支流程

2A1、若当日快速领用该笔订单下的礼品卡的验证短信发送次数已经超过十条，则提示用户：当日短信发送过于频繁，请明天重试

3A1、若用户验证码错误，统一验证码错误前四次，可以重新尝试输入，若同一验证码第五次错误，该条验证码失效，并发送一条新的短信验证码（如果该条短信为当日第十一条短信，也无法发送，提示内容同上）

异常流程

无

后置条件

该笔订单下的电子饭卡领用完毕后，再次进入订单列表页和订单详情页，不再展示【查看卡密】和【快速领用】按钮

业务规则

验证码通过验证后，有效时长为30分钟，超时之后验证状态失效，需要重新验证，并且验证状态与快速领用共用

约束

1、验证短信每日最多发送十条的限制仅针对同一笔订单，多笔订单可以超过十条短信

2、每条短信验证码有效时长为30分钟，从发送后开始计算，超时后验证码失效

备注

查看卡号密码的验证状态可以和快速领用共用，但不能和确认收货的验证状态共用
